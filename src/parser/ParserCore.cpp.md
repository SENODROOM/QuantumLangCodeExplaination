# ParserCore.cpp

## Overview

`ParserCore.cpp` is a crucial component of the Quantum Language compiler, responsible for parsing the input source code into an Abstract Syntax Tree (AST). This file contains the implementation of the `Parser` class, which handles the lexical analysis and syntactic parsing phases of the compilation process.

## Role in Compiler Pipeline

The parser's primary role is to convert the sequence of tokens generated by the lexer (`src/tokenizer/Tokenizer.cpp`) into a structured AST. The AST represents the syntactic structure of the program and serves as the foundation for subsequent stages of compilation, such as semantic analysis, optimization, and code generation.

## Key Design Decisions and Why

1. **Token Stream Management**:
   - The `Parser` constructor takes ownership of a vector of `Token`s, ensuring efficient memory management and preventing dangling references.
   
2. **Current Token Access**:
   - The `current()` method returns the token at the current position in the stream, facilitating easy access to the current token during parsing operations.
   
3. **Peeking Ahead**:
   - The `peek(int offset)` method allows the parser to look ahead in the token stream without advancing the position. This is essential for lookahead parsing techniques used to handle complex grammatical structures.
   
4. **Consuming Tokens**:
   - The `consume()` method advances the parser's position to the next token, returning the consumed token. This ensures that each token is processed exactly once.
   
5. **Expectation Handling**:
   - The `expect(TokenType t, const std::string &msg)` method checks if the current token matches the expected type. If not, it throws a `ParseError`, providing clear feedback on what went wrong and where.
   
6. **Type Checking and Matching**:
   - The `check(TokenType t) const` method simply checks if the current token matches the specified type.
   - The `match(TokenType t)` method combines checking and consuming the token, making it easier to handle optional tokens or sequences.
   
7. **End-of-File Detection**:
   - The `atEnd()` method determines whether the parser has reached the end of the token stream, allowing clean termination of the parsing process.
   
8. **Skipping Newlines**:
   - The `skipNewlines()` method efficiently skips over any newline tokens, ensuring that they do not interfere with the syntactic analysis of the program.

## Major Classes/Functions Overview

- **current()**: Returns the current token being processed.

- **expect(TokenType t, const std::string &msg)**: Checks if the current token matches the expected type and throws an error if it does not.
- **check(TokenType t)**: Checks if the current token matches the specified type.
- **match(TokenType t)**: Combines checking and consuming the token, handling optional tokens more gracefully.
- **atEnd()**: Determines if the parser has reached the end of the token stream.
- **skipNewlines()**: Skips over all consecutive newline tokens.
- **parse()**: Parses the entire token stream into an AST, starting with a block statement and processing individual statements until the end of the stream is reached.

## Tradeoffs

- **Memory Efficiency**: By taking ownership of the token vector in the constructor, the parser avoids unnecessary copies and reduces memory usage.
- **Flexibility**: The use of peeking and matching allows the parser to handle complex grammatical rules and make decisions based on future tokens.
- **Error Handling**: Throwing exceptions for parse errors provides immediate feedback and helps in debugging the source code.
- **Performance**: Efficiently skipping newlines minimizes overhead and improves performance, especially when dealing with large source files.

In summary, `ParserCore.cpp` plays a vital role in the Quantum Language compiler by converting raw tokens into a structured AST, enabling further compilation steps. Its design choices balance memory efficiency, flexibility, error handling, and performance, making it a robust and effective part of the compiler pipeline.
