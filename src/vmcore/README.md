# vmcore

The `vmcore` component is a critical module within the Quantum Language compiler, designed to execute bytecode and manage runtime operations efficiently. This directory houses several well-documented functions and files that collectively form the heart of the virtual machine, ensuring its state remains intact and facilitating seamless execution.

## Overview

The primary function of the `vmcore` component is to interpret and execute bytecode generated by the Quantum Language compiler. It manages various aspects of the runtime environment, including memory allocation, garbage collection, and error handling. The component consists of multiple interconnected files and functions that work harmoniously to achieve these objectives.

### Key Functions

- **run**: Initiates the execution of the bytecode.
- **push**: Adds an item to the stack.
- **pop**: Removes an item from the stack.
- **runtimeError**: Handles runtime errors gracefully.
- **toNumber**: Converts values to numbers.
- **valuesEqual**: Compares two values for equality.
- **execBinary**: Executes binary operations.
- **execUnary**: Executes unary operations.
- **captureUpvalue**: Captures upvalues during closure creation.
- **closeUpvalues**: Closes upvalues when a function exits.
- **callValue**: Calls a value as a function or method.
- **callClosure**: Calls a closure.
- **callNativeFn**: Invokes a native function.
- **callClass**: Instantiates a class and calls its constructor.
- **callBuiltinMethod**: Calls a built-in method on an object.

## File Structure

The `vmcore` directory is organized into several files, each focusing on specific functionalities:

- **vm.h**: Header file containing declarations for all public functions and data structures used by the `vmcore`.
- **vm.cpp**: Source file implementing the core logic of the virtual machine, including bytecode interpretation and stack management.
- **errors.h** and **errors.cpp**: Files dedicated to handling runtime errors, providing detailed messages and mechanisms for recovery.
- **memory.h** and **memory.cpp**: Manage memory allocation and deallocation, including garbage collection strategies.
- **stack.h** and **stack.cpp**: Define the stack data structure and associated operations like push and pop.
- **operations.h** and **operations.cpp**: Implement various operations such as binary and unary calculations, comparisons, and conversions.
- **closures.h** and **closures.cpp**: Handle closure creation, capture, and invocation.
- **classes.h** and **classes.cpp**: Manage class instantiation and method calling.
- **builtin_methods.h** and **builtin_methods.cpp**: Provide implementations for built-in methods that can be invoked directly on objects.

## Overall Flow

1. **Initialization**: The virtual machine is initialized, setting up necessary data structures and preparing for execution.
2. **Bytecode Interpretation**: The `run` function begins interpreting the bytecode, invoking appropriate functions based on the instructions encountered.
3. **Stack Operations**: Values are pushed onto and popped from the stack using the `push` and `pop` functions, respectively, to manage operands and results.
4. **Execution of Operations**: Binary and unary operations are executed using the `execBinary` and `execUnary` functions, which handle the computation and result storage.
5. **Handling Errors**: Runtime errors are detected and handled using the `runtimeError` function, ensuring the program can recover or terminate gracefully.
6. **Memory Management**: Memory allocation and deallocation are managed through the `memory` module, including garbage collection to reclaim unused memory.
7. **Closure Handling**: Closures are captured and closed appropriately using the `captureUpvalue` and `closeUpvalues` functions, maintaining the correct scope and lifetime of variables.
8. **Class and Method Invocation**: Classes are instantiated and methods are called using the `callClass` and `callBuiltinMethod` functions, allowing for object-oriented programming capabilities.
9. **Finalization**: After executing the bytecode, the virtual machine finalizes any remaining tasks, such as closing open upvalues and cleaning up resources.

By understanding and utilizing the components provided in the `vmcore` directory, developers can effectively create and manage quantum programs, leveraging the powerful features of the Quantum Language compiler.
