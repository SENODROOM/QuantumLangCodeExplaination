# vmcore

The `vmcore` component is a critical module within the Quantum Language compiler, designed to execute bytecode and manage runtime operations efficiently. This directory houses a suite of meticulously documented functions and files that collectively constitute the heart of the virtual machine, maintaining its operational integrity and facilitating seamless bytecode execution.

## Overview

The primary function of the `vmcore` component is to interpret and execute bytecode generated by the compiler. It manages various aspects of the runtime environment, including memory allocation, garbage collection, and error handling. The component's architecture is modular, with each file dedicated to specific functionalities:

- **run.cpp**: Contains the main entry point for the virtual machine, initiating the execution process.
- **stack.cpp**: Manages the stack data structure used for storing local variables and function call frames.
- **errors.cpp**: Handles runtime errors, providing detailed messages and mechanisms for recovery or termination.
- **conversion.cpp**: Converts values between different types, ensuring type safety during execution.
- **comparison.cpp**: Implements comparison logic for values, essential for conditional statements and loops.
- **binaryOps.cpp**: Executes binary operations on values, such as addition, subtraction, multiplication, etc.
- **unaryOps.cpp**: Performs unary operations on values, like negation or increment/decrement.
- **upvalues.cpp**: Manages upvalues, which are references to outer scope variables captured by closures.
- **gc.cpp**: Implements the garbage collection mechanism to reclaim unused memory.
- **calls.cpp**: Handles function calls, both native and user-defined, ensuring proper argument passing and return value management.
- **classes.cpp**: Manages class definitions and instances, supporting object-oriented programming features.
- **builtins.cpp**: Provides access to built-in methods and functions, enhancing the language's functionality without requiring additional code.

These components work in concert to create a robust and efficient virtual machine capable of running quantum programs effectively. The overall flow begins with the `run()` function, which initializes the virtual machine state and starts interpreting the bytecode. As the program executes, it utilizes the stack for temporary storage, performs comparisons and conversions using the respective modules, and handles function calls through the `calls.cpp` file. Error management is handled by `errors.cpp`, while garbage collection ensures memory efficiency. Object-oriented features are managed by `classes.cpp`, and built-in capabilities are provided by `builtins.cpp`.

## Files and Functions

### run.cpp
- **Function**: `run()`
- **Description**: Initializes the virtual machine and starts interpreting bytecode.

### stack.cpp
- **Functions**: `push()`, `pop()`
- **Description**: Manages the stack operations required for function calls and local variable storage.

### errors.cpp
- **Function**: `runtimeError()`
- **Description**: Handles runtime errors, logging them and potentially terminating the program.

### conversion.cpp
- **Function**: `toNumber()`
- **Description**: Converts values to numbers, essential for arithmetic operations.

### comparison.cpp
- **Function**: `valuesEqual()`
- **Description**: Compares two values for equality, used in conditional statements.

### binaryOps.cpp
- **Function**: `execBinary()`
- **Description**: Executes binary operations on two operands, such as addition, subtraction, etc.

### unaryOps.cpp
- **Function**: `execUnary()`
- **Description**: Executes unary operations on a single operand, like negation.

### upvalues.cpp
- **Function**: `captureUpvalue()`, `closeUpvalues()`
- **Description**: Manages upvalues, capturing and closing references to outer scope variables.

### gc.cpp
- **Function**: `collectGarbage()`
- **Description**: Initiates the garbage collection process to free up unused memory.

### calls.cpp
- **Functions**: `callValue()`, `callClosure()`, `callNativeFn()`, `callClass()`, `callBuiltinMethod()`
- **Description**: Handles all types of function calls, including native functions and methods defined in classes.

### classes.cpp
- **Function**: `defineClass()`, `createInstance()`
- **Description**: Manages class definitions and instance creation, supporting object-oriented programming.

### builtins.cpp
- **Function**: `registerBuiltins()`
- **Description**: Registers built-in methods and functions, making them available to the programmer.

Each file in the `vmcore` directory is thoroughly documented, detailing its purpose, implementation, and interactions with other components. This documentation serves as a valuable resource for developers working on the Quantum Language compiler, ensuring they understand how each part contributes to the overall system.

## Conclusion

The `vmcore` component is a vital component of the Quantum Language compiler, responsible for executing bytecode and managing runtime operations efficiently. By leveraging the modular design and comprehensive documentation provided in this directory, developers can gain a deep understanding of how the virtual machine operates and make informed decisions when extending or modifying the compiler.
