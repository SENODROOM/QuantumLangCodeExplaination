# LexerReaders.cpp

## Role in Compiler Pipeline

The `LexerReaders.cpp` file plays a critical role in the Quantum Language compiler's lexical analysis phase. Specifically, it focuses on parsing template literals, which are strings capable of containing embedded expressions enclosed in `${}`. This feature enables dynamic content within string literals, a common requirement in many programming languages.

## Key Design Decisions and Why

1. **Handling Escaped Characters**: The lexer must correctly interpret escaped characters such as `\n`, `\t`, `` ` `` (backtick), and others. To achieve this, the code checks for escape sequences and processes them accordingly before adding the character to the current segment.

2. **Expression Parsing**: Template literals include embedded expressions that need to be parsed separately. The lexer identifies these expressions by looking for `${` followed by any valid expression syntax until a matching `}` is found. Each identified expression is treated as a separate part to ensure accurate parsing.

3. **Tokenization**: After collecting all parts of the template literal, the lexer tokenizes each segment. Text segments become `STRING` tokens, while expression segments are re-lexed and placed within parentheses. This approach ensures that the expressions are properly interpreted during subsequent compilation stages.

4. **Efficiency**: The lexer efficiently handles the parsing process by iterating through the input source code once. It uses simple state transitions and conditional checks to identify and process different types of segments.

## Major Classes/Functions Overview

### Lexer Class

The `Lexer` class contains the primary logic for reading and processing the input source code. It includes methods like `readTemplateLiteral`, which is responsible for handling template literals.

#### readTemplateLiteral Function

This function parses template literals and populates the output vector with tokens. It skips the opening backtick and collects alternating text segments and expression sources. Each text segment becomes a `STRING` token, and each expression source is re-lexed and wrapped in parentheses.

### Token Class

The `Token` class represents individual tokens generated by the lexer. It includes properties such as `type`, `value`, `line`, and `column`.

### Error Handling

The `Lexer` class also includes error handling mechanisms to manage issues encountered during the parsing process, such as unmatched backticks or invalid escape sequences.

## Tradeoffs

1. **Complexity vs. Accuracy**: By treating each part of the template literal separately, the lexer increases its complexity but ensures higher accuracy in parsing both text and embedded expressions.

2. **Performance vs. Simplicity**: The single-pass nature of the lexer provides good performance, but it requires more intricate logic to handle the various cases of template literals.

3. **Flexibility vs. Strictness**: Allowing dynamic content within string literals enhances flexibility, but it may introduce complexity in error handling and validation.

Overall, the `LexerReaders.cpp` file is an essential component of the Quantum Language compiler, providing robust support for template literals while maintaining efficiency and accuracy in the lexical analysis phase.
