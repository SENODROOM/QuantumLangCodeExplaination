# `emitStore` Function

## Overview

The `emitStore` function is a crucial method within the Quantum Language compiler's core functionality. Its primary role is to generate bytecode instructions that store values into variables or fields. This function ensures that the correct type of storage instruction is emitted based on whether the variable is local, upvalue, or global.

## Parameters and Return Value

- **Parameters**:
  - `name`: A string representing the name of the variable or field to be stored.
  - `line`: An integer indicating the source code line number where the store operation occurs.

- **Return Value**: 
  - The function does not explicitly return any value but generates bytecode instructions directly through calls to `emit`.

## Detailed Explanation

### Step-by-Step Execution

1. **Name Resolution**:
   ```cpp
   const std::string &resolved = (name == "this") ? std::string("self") : name;
   ```
   This step checks if the variable name is `"this"`. If so, it resolves it to `"self"` because in Quantum Language, `"this"` is often used as a shorthand for the current object instance, which is referred to as `"self"` in the generated bytecode.

2. **Local Variable Check**:
   ```cpp
   int local = resolveLocal(current_, resolved);
   if (local != -1)
   {
       emit(Op::STORE_LOCAL, local, line);
       return;
   }
   ```
   It then attempts to find the index of the variable in the current scope using the `resolveLocal` function. If the variable is found (`local != -1`), it emits a `STORE_LOCAL` bytecode instruction with the variable's index and the source code line number. This instruction stores the value onto the stack at the specified local slot.

3. **Upvalue Check**:
   ```cpp
   int uv = resolveUpvalue(current_, resolved);
   if (uv != -1)
   {
       emit(Op::STORE_UPVALUE, uv, line);
       return;
   }
   ```
   If the variable is not found locally, the function checks if it is an upvalue (a variable from an enclosing scope) using the `resolveUpvalue` function. If an upvalue is found (`uv != -1`), it emits a `STORE_UPVALUE` bytecode instruction with the upvalue's index and the source code line number. This instruction stores the value onto the stack at the specified upvalue slot.

4. **Global Variable Check**:
   ```cpp
   emit(Op::STORE_GLOBAL, addStr(resolved), line);
   ```
   If neither a local nor an upvalue is found, the function assumes the variable is global. It adds the global variable name to the string table using `addStr(resolved)` and emits a `STORE_GLOBAL` bytecode instruction with the global variable's index and the source code line number. This instruction stores the value onto the stack at the specified global slot.

### Edge Cases

- **Local Variables**: When storing a local variable, ensure that the variable has been declared and initialized before the store operation.
- **Upvalues**: Upvalues are typically used in nested functions. Ensure that the outer function's variables are accessible and correctly captured by the inner function.
- **Global Variables**: Global variables should be declared in the global scope. Storing a non-declared global variable will result in a runtime error.

### Interactions with Other Components

- **String Table (`addStr`)**: The `addStr` function interacts with the string table to manage unique string identifiers. This ensures efficient memory usage and quick lookups for global variable names.
  
- **Bytecode Emission (`emit`)**: The `emit` function is responsible for generating and emitting bytecode instructions. It takes the opcode, operands, and line number as arguments and appends them to the bytecode buffer.

- **Scope Resolution (`resolveLocal`, `resolveUpvalue`)**: These functions interact with the current scope and its enclosing scopes to determine the location of the variable. They help in managing variable lifetimes and access rules within different scopes.

In summary, the `emitStore` function is a vital component of the Quantum Language compiler that handles the storage of values into variables or fields. By resolving the variable's location and emitting the appropriate bytecode instruction, it ensures efficient and accurate compilation of the source code.
