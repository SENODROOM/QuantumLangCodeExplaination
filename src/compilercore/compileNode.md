# `compileNode` Function

## Purpose
The `compileNode` function is a crucial part of the Quantum Language compiler, designed to process and compile individual nodes within the abstract syntax tree (AST). Each node corresponds to a distinct construct in the source code, including statements, expressions, and declarations. The primary role of `compileNode` is to translate these high-level constructs into low-level quantum machine instructions that can be executed on a quantum computer.

## Parameters
- `node`: A reference to an AST node that needs to be compiled. This parameter is polymorphic, meaning it can represent any type of AST node, such as a statement, expression, or declaration.

## Return Value
This function does not return a value explicitly. Instead, it performs actions directly related to the compilation process, such as emitting quantum operations and handling variable declarations.

## Edge Cases
1. **Import Statements**: When encountering an import statement (`ImportStmt`), `compileNode` delegates the task to native handlers since the compiler itself does not manage external libraries or modules.
2. **Empty Nodes**: If the node being processed is empty or undefined, `compileNode` may throw an exception or simply ignore it, depending on the implementation details.
3. **Unsupported Node Types**: For any node types that are not currently supported by the compiler, `compileNode` should either log a warning or error message indicating the unsupported nature of the node.

## Interactions with Other Components
- **Abstract Syntax Tree (AST)**: The `compileNode` function operates on nodes extracted from the AST generated by the parser. It uses the structure and data contained within these nodes to determine how to compile them.
- **Quantum Operation Emitter**: Within `compileNode`, various helper functions like `compileBlock`, `compileVarDecl`, etc., call upon the `emit` function to generate quantum machine instructions. These instructions are then added to the quantum program being compiled.
- **Error Handling**: The function includes conditional checks to handle different types of nodes. Unsupported node types trigger error messages, ensuring that only valid constructs are processed during compilation.
- **Scope Management**: During the compilation of variable declarations (`VarDecl`) and function declarations (`FunctionDecl`), scope management is handled. This ensures that variables and functions are correctly defined and accessible within their respective scopes.

## Implementation Details
The `compileNode` function utilizes C++'s `std::visit` to dispatch the compilation logic based on the type of the node. This approach allows for a clean separation of concerns, where each node type has its own dedicated compilation function (`compileBlock`, `compileVarDecl`, etc.). Here's a breakdown of how it works:

1. **Line Number Extraction**: The line number associated with the current node is extracted (`int ln = node.line;`). This information is crucial for generating accurate error messages and debugging information.
2. **Polymorphic Dispatch**: Using `std::visit`, the function dynamically dispatches to the appropriate node-specific handler based on the type of the node.
3. **Conditional Compilation**: For each node type, there is a corresponding conditional block that calls the relevant compilation function. For example:
   ```cpp
   else if constexpr (std::is_same_v<T, VarDecl>) compileVarDecl(n, ln);
   ```
4. **Expression Compilation**: For expression nodes (`ExprStmt`), the function compiles the expression and then emits a `POP` operation to discard the result, assuming it's not used elsewhere in the code.
5. **Default Case**: In case the node type is not explicitly handled, the function falls back to generic expression compilation followed by a `POP` operation.

This design ensures that `compileNode` is both flexible and efficient, capable of handling a wide range of quantum language constructs while maintaining clear and manageable code structure.
