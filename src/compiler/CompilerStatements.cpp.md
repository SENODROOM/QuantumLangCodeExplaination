# Quantum Language Compiler: Statements Compilation

## Overview

The `src/compiler/CompilerStatements.cpp` file is an essential part of the Quantum Language compiler that handles the compilation of various types of statements such as variable declarations, function declarations, and class declarations. It converts high-level quantum language code into executable bytecode, ensuring efficient execution within the virtual machine.

## Role in Compiler Pipeline

This file operates during the intermediate code generation stage of the compiler pipeline. After parsing the source code into an abstract syntax tree (AST), it traverses the AST to generate bytecode instructions. The generated bytecode is then executed by the virtual machine (`Vm`) to perform operations defined in the quantum language.

## Key Design Decisions and Why

### Scope Management

- **Global vs Local Variables**: The compiler differentiates between global and local variables based on the scope depth. Global variables are stored using `Op::DEFINE_GLOBAL`, while local variables use `Op::DEFINE_LOCAL`.
- **Why**: This approach ensures that variables are correctly scoped and accessible at runtime, preventing conflicts and unintended side effects.

### Variable Declarations

- **Initialization Handling**: If a variable declaration includes an initializer, the expression is compiled. Otherwise, `Op::LOAD_NIL` is emitted to load a nil value.
- **Why**: Initialization allows for setting default values or expressions, enhancing the flexibility and readability of the code.

### Function Declarations

- **Closure Creation**: Functions can define closures, which capture variables from their enclosing scopes. The compiler generates bytecode to create these closures.
- **Why**: Closures enable functions to maintain state across multiple invocations, making them powerful tools for functional programming.

### Class Declarations

- **Inheritance Support**: Classes can inherit from other base classes. The compiler emits bytecode to handle inheritance, allowing derived classes to access methods and properties of their parent class.
- **Why**: Inheritance promotes code reuse and organization, enabling developers to build complex systems with modular components.

## Major Classes/Functions Overview

### `Compiler`

- **Overview**: The main compiler class that manages the compilation process, including statement compilation.
- **Key Methods**:
  - `compileVarDecl`: Compiles variable declarations.
  - `compileFunctionDecl`: Compiles function declarations.
  - `compileClassDecl`: Compiles class declarations.

### `VarDecl`

- **Overview**: Represents a variable declaration in the AST.
- **Attributes**:
  - `name`: The name of the variable.
  - `initializer`: An optional expression used to initialize the variable.
  - `isConst`: Indicates whether the variable is constant.

### `FunctionDecl`

- **Overview**: Represents a function declaration in the AST.
- **Attributes**:
  - `name`: The name of the function.
  - `params`: A list of parameters for the function.
  - `paramIsRef`: Indicates whether each parameter is passed by reference.
  - `defaultArgs`: Default arguments for the function.
  - `body`: The body of the function.

### `ClassDecl`

- **Overview**: Represents a class declaration in the AST.
- **Attributes**:
  - `name`: The name of the class.
  - `base`: The name of the base class, if any.
  - `members`: A list of members (variables, functions, etc.) within the class.

## Tradeoffs

- **Memory Usage**: Storing variable names and constants in the virtual machine requires additional memory. However, this tradeoff is necessary for efficient bytecode execution.
- **Execution Speed**: By emitting specific bytecode instructions for different operations (e.g., `Op::LOAD_CONST` for loading constants), the compiler optimizes execution speed.
- **Code Readability**: While the bytecode format may be less readable than high-level source code, it provides a clear, structured representation of program operations, facilitating debugging and maintenance.

Overall, the `src/compiler/CompilerStatements.cpp` file plays a critical role in transforming quantum language code into executable bytecode, supporting advanced features like closures and inheritance while managing memory and execution efficiently.
