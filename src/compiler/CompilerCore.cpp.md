# Quantum Language Compiler Core

## Overview

`src/compiler/CompilerCore.cpp` is a crucial component of the Quantum Language compiler, responsible for converting Abstract Syntax Trees (ASTs) into executable bytecode. This file contains the core logic for compiling different types of nodes within the AST and managing the compilation process's state.

### Role in Compiler Pipeline

The `CompilerCore.cpp` plays a central role in the compilation pipeline:
- **Parsing**: Converts source code into an AST.
- **Compilation**: Transforms the AST into bytecode that can be executed by the Quantum Virtual Machine (QVM).
- **Optimization**: Optionally applies optimizations to the generated bytecode.
- **Execution**: Runs the optimized bytecode on the QVM.

## Key Design Decisions and Why

### Scope Management

Scopes are managed using nested structures (`CompilerState`). Each scope has its own depth and a list of local variables. The `beginScope()` and `endScope()` functions handle entering and exiting scopes, respectively. When exiting a scope, it pops locals until it reaches the current scope depth, emitting appropriate bytecode operations (`POP` or `CLOSE_UPVALUE`) based on whether the variable was captured.

**Why**: This approach allows for dynamic scope resolution at runtime, which is essential for languages with features like closures and function calls.

### Local Variable Resolution

Variables are resolved locally first before checking enclosing scopes. If a variable is found locally, it emits a `LOAD_LOCAL` operation. If not, it checks if the variable is an upvalue (a local from an enclosing scope). Upvalues are tracked separately and require special handling when accessing or modifying them.

**Why**: This ensures that local variables are accessed as efficiently as possible, reducing the overhead of scope lookups.

### Bytecode Emission

Bytecode is emitted through the `emit()` function, which takes an operation type and optional operands. Operations include loading values, storing values, and returning results. Special handling is provided for capturing variables in closures.

**Why**: Emitting bytecode directly from the compiler allows for efficient execution by the QVM, leveraging its internal optimizations.

## Major Classes/Functions Overview

### Compiler Class

- **Constructor**: Initializes the compiler with a default state.
- **compile(ASTNode &root)**: Compiles the entire AST starting from the root node. It handles block statements and individual nodes, ensuring that the final chunk ends with a `RETURN_NIL` operation.

### CompilerState Struct

- **Members**: 
  - `scopeDepth`: Tracks the current scope depth.
  - `locals`: Stores information about local variables.
  - `upvalues`: Tracks upvalues (variables from enclosing scopes).
  - `chunk`: Holds the compiled bytecode.
  - `enclosing`: Points to the enclosing scope state.

- **Methods**:
  - `beginScope()`: Increases the scope depth.
  - `endScope(int line)`: Decreases the scope depth and cleans up locals/upvalues.
  - `resolveLocal(const std::string &name)`: Resolves a variable within the current scope.
  - `addUpvalue(int index, bool isLocal)`: Adds an upvalue to the current scope.
  - `resolveUpvalue(const std::string &name)`: Resolves a variable within an enclosing scope and converts it to an upvalue.

### Helper Functions

- **declareLocal(const std::string &name, int)**: Declares a new local variable in the current scope.
- **emitLoad(const std::string &name, int line)**: Emits a load operation for a variable, resolving it either locally or via upvalues.
- **emitStore(const std::string &name, int line)**: Emits a store operation for a variable, similar to `emitLoad`.

## Tradeoffs

### Memory Usage

- **Pros**: Efficient memory usage by reusing existing scope states and avoiding deep copying.
- **Cons**: Potential stack overflow issues if the scope nesting is too deep.

### Performance

- **Pros**: Direct bytecode emission reduces the overhead of interpreting high-level language constructs.
- **Cons**: Complicates the compiler implementation due to the need for efficient scope management and upvalue tracking.

### Flexibility

- **Pros**: Allows for dynamic scope resolution and closure support.
- **Cons**: Can lead to more complex code and potential performance penalties if not handled carefully.

In summary, `src/compiler/CompilerCore.cpp` is a well-designed and critical part of the Quantum Language compiler, providing the necessary functionality to convert ASTs into executable bytecode while managing scopes and variables efficiently.
