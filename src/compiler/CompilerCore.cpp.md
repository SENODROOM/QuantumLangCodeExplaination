# Quantum Language Compiler Core

## Overview

`src/compiler/CompilerCore.cpp` serves as the heart of the Quantum Language compiler, tasked with transforming Abstract Syntax Trees (ASTs) into executable bytecode. This file houses the primary logic required to handle diverse AST nodes and maintain the efficiency of the compilation process.

### Role in Compiler Pipeline

In the broader context of the Quantum Language compiler's pipeline, `CompilerCore.cpp` operates at the crucial stage of code generation. It takes the parsed AST and iteratively processes each node, emitting corresponding bytecode instructions. The generated bytecode is then executed by the Virtual Machine (VM). Key responsibilities include:

- **AST Traversal**: Recursively traverses the AST to convert it into bytecode.
- **Bytecode Emission**: Emits bytecode instructions based on the type of AST node encountered.
- **Scope Management**: Manages variable scopes, ensuring that variables are correctly referenced and captured within nested functions.
- **Error Handling**: Intercepts and handles errors during the compilation process, providing meaningful feedback to the user.

### Key Design Decisions and Why

#### Scope Management

**Why**: Efficiently managing variable scopes is essential for correct variable resolution and capturing within closures. By maintaining a stack of `CompilerState` objects, each representing a scope level, we can easily track where variables are declared and accessed.

**Implementation**: The `CompilerState` class contains a vector of `Local` objects, which store information about each local variable, including its name, depth, and whether it has been captured. The `declareLocal`, `emitLoad`, and `emitStore` functions use these structures to ensure proper handling of local and global variables.

#### Upvalue Resolution

**Why**: Upvalues are used to capture variables from outer scopes in nested functions. Resolving upvalues ensures that variables are correctly captured and accessed when the inner function is executed.

**Implementation**: The `resolveUpvalue` function searches for a variable in the enclosing scope and marks it as captured. If the variable is found, it returns an index to an upvalue structure stored in the current scope. The `addUpvalue` function adds new upvalues to the current scope and increments the upvalue count in the chunk.

#### Error Handling

**Why**: Robust error handling is critical for debugging and improving the reliability of the compiler. By intercepting errors and providing detailed messages, developers can quickly identify and fix issues in their source code.

**Implementation**: The `Compiler` class includes error handling mechanisms such as `reportError` and `errorAt`. These functions log errors and provide context-specific information, aiding in effective debugging.

### Major Classes/Functions Overview

#### Compiler Class

**Overview**: The `Compiler` class is the main entry point for the compilation process. It initializes the compilation environment, manages the state of the compilation process, and emits bytecode instructions.

**Key Functions**:
- `compile`: Converts the root AST node into bytecode.
- `emit`: Emits a single bytecode instruction.
- `emitLoad`: Loads a variable from the local or global scope.
- `emitStore`: Stores a value into a variable in the local or global scope.

#### CompilerState Class

**Overview**: Represents the state of the compilation process at a given scope level. Each `CompilerState` object contains a chunk of bytecode, a list of local variables, and a list of upvalues.

**Key Members**:
- `chunk`: Holds the generated bytecode.
- `locals`: Tracks local variables and their depths.
- `upvalues`: Tracks upvalues for capturing variables from outer scopes.

#### Local Struct

**Overview**: Represents a local variable within a scope. Contains the variable's name, depth, and whether it has been captured.

**Key Members**:
- `name`: Variable name.
- `depth`: Depth of the variable in the scope stack.
- `isCaptured`: Indicates whether the variable is captured in an upvalue.

#### Upvalue Struct

**Overview**: Represents an upvalue, which captures a variable from an outer scope. Contains the index of the captured variable and whether it is local.

**Key Members**:
- `index`: Index of the captured variable in the enclosing scope's locals.
- `isLocal`: Indicates whether the captured variable is local to the enclosing scope.

### Tradeoffs

#### Memory Usage vs. Performance

**Memory Usage**: Maintaining a stack of `CompilerState` objects requires additional memory to store scope information.

**Performance**: Efficient scope management and upvalue resolution enhance performance by reducing the need for repeated searches and minimizing the overhead associated with variable access.

By carefully balancing these factors, the Quantum Language compiler ensures both efficient and reliable code generation.
