# Quantum Language Compiler Core

## Overview

`src/compiler/CompilerCore.cpp` serves as the heart of the Quantum Language compiler, transforming Abstract Syntax Trees (ASTs) into executable bytecode. This file encapsulates the essential logic for compiling various AST nodes and managing the state of the compilation process.

### Role in Compiler Pipeline

The `CompilerCore.cpp` plays a pivotal role in the compiler's pipeline:
- **Parsing**: It receives an AST generated by the parser.
- **Compilation**: It traverses the AST to generate bytecode instructions.
- **Optimization**: It manages scope depths and resolves variables to their correct locations.
- **Output**: It finalizes the bytecode with a return instruction and returns a chunk containing the compiled code.

This ensures that each part of the AST is handled appropriately, resulting in efficient and error-free bytecode generation.

### Key Design Decisions and Why

#### 1. Scope Management
- **Design Decision**: The compiler maintains a stack of scopes (`CompilerState`) to track variable declarations and captures.
- **Why**: This allows for dynamic scope resolution and management, which is crucial for handling nested functions and variable lifetimes.

#### 2. Variable Resolution
- **Design Decision**: The compiler uses a combination of local and upvalue resolution to locate variables.
- **Why**: This approach optimizes access to variables by reducing redundant searches and minimizing the number of operations needed during execution.

#### 3. Bytecode Emission
- **Design Decision**: Instructions are emitted using a simple interface that handles both local and global variable loads/stores.
- **Why**: This abstraction simplifies the emission of bytecode and makes it easier to extend or modify without impacting the core logic.

### Major Classes/Functions Overview

#### Compiler Class
- **Overview**: Manages the overall compilation process, including parsing, emitting bytecode, and maintaining scope states.
- **Functions**:
  - `compile(ASTNode &root)`: Compiles the root node of the AST and returns the resulting chunk.
  - `emit(Op op, int arg1, int arg2)`: Emits a single bytecode instruction.
  - `emitLoad(const std::string &name, int line)`: Emits a load instruction for a variable.
  - `emitStore(const std::string &name, int line)`: Emits a store instruction for a variable.

#### CompilerState Struct
- **Overview**: Represents the state of the compiler at any given point, including the current scope depth and list of locals/upvalues.
- **Members**:
  - `scopeDepth`: Tracks the current scope depth.
  - `locals`: Stores information about local variables within the current scope.
  - `upvalues`: Stores information about captured variables from enclosing scopes.

#### Helper Functions
- `beginScope()`: Increments the scope depth.
- `endScope(int line)`: Decrements the scope depth and cleans up local/upvalue references.
- `resolveLocal(CompilerState *state, const std::string &name)`: Resolves a local variable by its name.
- `addUpvalue(CompilerState *state, int index, bool isLocal)`: Adds an upvalue reference to the current scope.
- `resolveUpvalue(CompilerState *state, const std::string &name)`: Resolves an upvalue by its name.
- `declareLocal(const std::string &name, int line)`: Declares a new local variable in the current scope.

### Tradeoffs

- **Complexity vs. Performance**: Maintaining a stack-based scope system adds complexity but improves performance by allowing quick lookups and minimal overhead.
- **Memory Usage**: Storing local and upvalue information can lead to increased memory usage, especially in deeply nested functions.
- **Extensibility**: The use of a simple interface for bytecode emission makes the compiler more extensible, but may require additional logic for complex scenarios.

Overall, `src/compiler/CompilerCore.cpp` is a well-designed and optimized component of the Quantum Language compiler, ensuring efficient and accurate bytecode generation while managing the complexities of scope and variable resolution.
