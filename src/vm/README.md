# VM Component of the Quantum Language Compiler

The VM (Virtual Machine) component is a crucial part of the Quantum Language compiler, responsible for executing bytecode generated by the compiler. This component is designed to be highly efficient and flexible, allowing for the execution of various quantum operations and control structures.

## Overview

The VM operates on a stack-based architecture, where instructions manipulate values stored in the stack. It handles different types of operations such as arithmetic, logical, control flow, and function calls. The component includes several key functionalities:

- **Bytecode Execution**: Interprets and executes bytecode instructions.
- **Stack Management**: Manages the stack for storing operands and intermediate results.
- **Runtime Error Handling**: Detects and handles runtime errors gracefully.
- **Function Calls**: Supports both native and user-defined function calls.
- **Control Flow**: Implements control flow constructs like loops and conditionals.

## Files and Functions

This directory contains several documented functions and files that contribute to the VM's functionality:

### `vm.cpp` and `vm.h`

These files define the core classes and functions for the VM. They include:

- `class VM`: Represents the virtual machine itself, managing the stack and bytecode execution.
- `void run()`: Main function to start the VM and execute the bytecode.
- `void push(Value value)`: Pushes a value onto the stack.
- `Value pop()`: Pops a value from the stack.
- `void runtimeError(const std::string& message)`: Handles runtime errors by printing an error message and exiting the program.

### `operations.cpp` and `operations.h`

These files contain implementations for various bytecode operations:

- `void execBinary(Instruction instruction)`: Executes binary operations like addition or multiplication.
- `void execUnary(Instruction instruction)`: Executes unary operations like negation or increment.
- `void captureUpvalue(int index)`: Captures upvalues for closures.
- `void closeUpvalues(Value* lastClosed)`: Closes upvalues when a closure is called.
- `void callValue(Value callable, int argumentCount)`: Calls a user-defined function or method.
- `void callClosure(Closure* closure, int argumentCount)`: Calls a closure.
- `void callNativeFn(NativeFunction fn, int argumentCount)`: Calls a native function.
- `void callClass(Class* klass, int argumentCount)`: Calls a constructor for a class.
- `void callBuiltinMethod(BuiltinMethod method, Value receiver, int argumentCount)`: Calls a built-in method on a receiver.
- `void callArrayMethod(Array* array, BuiltinMethod method, int argumentCount)`: Calls a method on an array.
- `void callStringMethod(String* string, BuiltinMethod method, int argumentCount)`: Calls a method on a string.
- `void callDictMethod(Dict* dict, BuiltinMethod method, int argumentCount)`: Calls a method on a dictionary.

### `values.cpp` and `values.h`

These files handle value manipulation and type conversions:

- `double toNumber(Value value)`: Converts a value to a number.
- `bool valuesEqual(Value left, Value right)`: Compares two values for equality.

## Overall Flow

1. **Initialization**: The VM is initialized with an empty stack and the bytecode to be executed.
2. **Bytecode Interpretation**: Instructions are read one by one from the bytecode stream.
3. **Operand Stack Management**: Operands are pushed onto the stack during the interpretation process.
4. **Operation Execution**: Depending on the instruction, the corresponding operation function is called to perform the operation.
5. **Result Handling**: Results of operations are either used immediately or pushed back onto the stack for further processing.
6. **Control Flow**: Control flow instructions like jumps and branches are handled to manage the execution flow.
7. **Function Calls**: Function calls are made using the appropriate function based on whether it's a native function, user-defined function, or method call.
8. **Error Handling**: Runtime errors are detected and handled by calling the `runtimeError` function.
9. **Termination**: Once all bytecode has been executed, the VM terminates.

This structured approach ensures that the VM can efficiently execute complex quantum programs while maintaining robustness and flexibility.
