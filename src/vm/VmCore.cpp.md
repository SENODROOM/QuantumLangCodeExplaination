# Quantum Language Compiler - VmCore.cpp

## Overview

`VmCore.cpp` is a vital component of the Quantum Language compiler's virtual machine (VM) core. It houses fundamental functions necessary for executing compiled code segments within the VM environment. This file plays a crucial role in the compiler pipeline, especially during the execution phase where generated bytecode is interpreted.

### Key Design Decisions and Why

1. **Iterator State Tagging**:
   - Encodes iterators using `QuantumNative` objects with a function that never gets called.
   - The VM identifies these iterators through a specific naming convention (`__iter__`) and stores their state in an `IterState` map.
   - **Why**: This approach allows for efficient identification and management of iterator states without altering the existing `QuantumNative` structure.

2. **Stack Management**:
   - Implements stack operations such as `push`, `pop`, and `peek`.
   - Ensures robust error handling for stack underflows.
   - **Why**: A well-managed stack is essential for maintaining correct execution contexts and passing data between different parts of the VM.

3. **Runtime Error Handling**:
   - Provides a mechanism to throw runtime errors with detailed messages and line numbers.
   - **Why**: Comprehensive error reporting aids in debugging and ensures that issues are addressed promptly.

4. **Type Conversion**:
   - Includes functionality to convert values between different types (e.g., string to number).
   - Throws appropriate exceptions when conversions fail.
   - **Why**: Type safety is crucial in any programming language, and this feature ensures that type mismatches are caught early and handled gracefully.

5. **Value Equality Check**:
   - Offers a method to compare two `QuantumValue` instances for equality.
   - Handles various types including nil, boolean, number, and string.
   - **Why**: Accurate value comparison is essential for control flow statements like conditional checks and loops.

### Major Classes/Functions Overview

- **VM Class**:
  - Manages the overall state of the VM, including global variables, stack, frames, and handlers.
  - **Constructor**: Initializes the VM with a new global environment and registers native functions.
  - **Run Method**: Executes a given chunk of code by setting up the initial frame and calling `runFrame`.

- **Stack Helpers**:
  - `push`: Adds a value to the stack.
  - `pop`: Removes the top value from the stack and returns it.
  - `peek`: Retrieves a value from the stack without removing it.
  - `runtimeError`: Throws a runtime error with a message and line number.
  - `toNumber`: Converts a value to a number, throwing an exception if conversion fails.

- **Value Equality Check**:
  - `valuesEqual`: Compares two `QuantumValue` instances for equality, considering different types.

### Tradeoffs

- **Performance vs. Memory Usage**:
  - Using dynamic memory allocation for the stack can lead to higher memory usage but offers better performance due to reduced overhead compared to fixed-size arrays.
  
- **Flexibility vs. Complexity**:
  - Implementing comprehensive type conversion and error handling adds complexity to the VM core but enhances its flexibility and robustness.

- **Readability vs. Efficiency**:
  - While the use of `std::move` in stack operations improves efficiency, it may slightly reduce readability for beginners.

Overall, `VmCore.cpp` is a foundational file in the Quantum Language compiler, providing essential functionalities for VM operation and ensuring the smooth execution of compiled code.
