# skipComment

The `skipComment` function is an essential part of the LexerCore class in the Quantum Language compiler. Its primary role is to efficiently skip over comment sections within the source code during lexical analysis. This allows the compiler to focus on parsing meaningful code rather than ignoring unnecessary commentary.

## What It Does

The `skipComment` function reads through the source code starting from its current position (`pos`) until it encounters a newline character (`'\n'`). During this process, it advances the position marker (`pos`) one character at a time using the `advance()` method. The function effectively ignores any characters between the start of a comment and the end of the line, treating them as non-code content.

## Why It Works This Way

This approach ensures that the lexer can quickly bypass comment sections without processing their contents, which would be redundant and potentially costly in terms of performance. By stopping at the first newline character encountered after the comment, the function minimizes the number of characters read and analyzed, focusing only on the relevant parts of the source code.

## Parameters/Return Value

- **Parameters**: None
- **Return Value**: None

The function operates directly on the internal state of the LexerCore object, modifying the `pos` member variable to indicate where the next token should begin. It does not return any values since its sole purpose is to update the lexer's state.

## Edge Cases

1. **Single-line Comments**: The function handles both single-line comments (e.g., `// This is a comment`) and multi-line comments (e.g., `/* This is a comment */`). For single-line comments, it stops at the first newline character. For multi-line comments, it continues reading until the closing `*/` sequence is found, but this is beyond the scope of the `skipComment` function itself.
2. **End-of-Line Comments**: If a comment appears at the very end of a line, the function will correctly stop at the newline character, ensuring that the lexer moves to the next line without attempting to parse any additional characters.
3. **Empty Lines**: The function gracefully handles empty lines, simply advancing the position marker until it reaches the newline character.

## Interactions With Other Components

The `skipComment` function interacts primarily with the LexerCore class and its associated methods. It relies on the `current()` method to retrieve the character at the current position (`pos`) and the `advance()` method to move the position marker forward. Additionally, it indirectly interacts with the TokenStream component, which consumes tokens generated by the lexer. By skipping comments, the lexer ensures that the TokenStream receives only valid code tokens, simplifying subsequent stages of compilation.
