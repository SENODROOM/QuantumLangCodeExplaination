# Disassembler.cpp

## Role in Compiler Pipeline

`Disassembler.cpp` is an essential component of the Quantum Language compiler, responsible for converting bytecode back into human-readable assembly-like instructions. This process, known as disassembly, aids in debugging, understanding, and optimizing the code generated by the compiler. The disassembler operates on the output produced by the bytecode generator, which translates high-level source code into a sequence of machine instructions that can be executed by the quantum virtual machine (QVM).

## Key Design Decisions and Why

1. **String-Based Representation**: The disassembler uses string-based representations for operations (`Op`). This decision simplifies the mapping between opcode values and their corresponding names, making it easier to read and understand the disassembled output.

2. **Efficient Lookup**: A `switch` statement is employed for efficient lookup of operation names based on the opcode value. This approach ensures fast execution time, crucial for performance-sensitive tools like debuggers.

3. **Stream Output**: The disassembler outputs the disassembled instructions to a stream (`std::ostream`). This flexibility allows the disassembled output to be directed to various destinations, such as standard output or log files, without modifying the core functionality of the disassembler.

4. **Bytecode Format Independence**: By using an enumeration type (`Op`) to represent operations, the disassembler remains independent of the specific format of the bytecode. This abstraction layer facilitates changes to the bytecode format without impacting the disassembling logic.

## Major Classes/Functions Overview

### Function: `opName(Op op)`

- **Purpose**: Maps an `Op` enum value to its corresponding string representation.
- **Parameters**:
  - `op`: An `Op` enum value representing a bytecode instruction.
- **Return Value**: A pointer to a constant character string (`const char*`) that describes the operation.
- **Usage**: This function is invoked within the disassembler loop to convert each opcode into a readable form before outputting it.

### Class: `Disassembler`

- **Purpose**: Manages the disassembling process, converting bytecode into human-readable instructions.
- **Key Methods**:
  - `disassemble(const std::vector<uint8_t>& bytecode)`: Main method that takes a vector of bytecode and disassembles it into a series of instructions.
  - `printInstruction(std::ostream& out, uint8_t opcode, size_t ip)`: Helper method to print a single instruction to the specified output stream.
- **Attributes**:
  - `std::vector<std::string> globalNames`: Stores the names of global variables used in the bytecode.
  - `std::vector<std::string> localNames`: Stores the names of local variables used in the bytecode.
- **Trade-offs**:
  - **Memory Usage**: Storing the names of global and local variables requires additional memory, but improves readability and maintainability of the disassembled output.
  - **Performance**: The use of `switch` statements for opcode lookup provides optimal performance, suitable for real-time debugging scenarios.

## Tradeoffs

- **Memory Overhead**: Maintaining separate vectors for global and local variable names consumes extra memory, which might not be a concern for small programs but could impact larger applications.
- **Readability vs. Performance**: While the string-based representation enhances readability, the use of `switch` statements ensures minimal runtime overhead, critical for tools that need to operate efficiently on large bytecode outputs.
