# QuantumLanguage Compiler - Parser.h

## Overview

The `include/Parser.h` header file is an essential part of the QuantumLanguage compiler's architecture, responsible for converting a sequence of tokens into an Abstract Syntax Tree (AST). This file plays a pivotal role in the compiler pipeline by handling the syntactic analysis phase, ensuring that the input source code adheres to the language's grammar rules. The AST generated by this parser serves as the foundation for subsequent phases of compilation, including semantic analysis, optimization, and code generation.

## Key Design Decisions

### Token Helpers

Key design decisions were made regarding token manipulation to ensure flexibility and readability. For instance, the use of reference parameters (`&`) in functions like `consume()` and `expect()` allows these functions to modify the current position in the token stream without copying the entire vector. This approach minimizes memory usage and enhances performance.

### Pratt Parsing Algorithm

The Pratt parsing algorithm was chosen for its ability to handle operator precedence and associativity effectively. By separating the parsing logic into different precedence levels, the algorithm can manage complex expressions seamlessly. This decision ensures that the compiler can correctly interpret nested operations and resolve them according to the specified rules, thereby maintaining the integrity and correctness of the program.

## Class and Function Documentation

### ParseError

**Purpose:** Represents errors encountered during the parsing process, providing additional context about the error location.

**Behaviour:** Inherits from `std::runtime_error`, adding line and column numbers to help pinpoint the exact location of the error in the source code.

### Parser

**Purpose:** Manages the parsing of the token stream into an AST.

**Behaviour:** Initializes with a vector of tokens and provides methods to parse statements, expressions, and other constructs. It uses a Pratt-style parsing algorithm to handle operator precedence and associativity.

#### Public Methods

- **explicit Parser(std::vector<Token> tokens):** Constructor initializes the parser with a list of tokens.
- **ASTNodePtr parse():** Main entry point for parsing, returns the root node of the AST.

#### Private Methods

- **Token &current():** Returns the current token being processed.
- **Token &peek(int offset = 1):** Returns the token at a given offset ahead in the token stream.
- **Token &consume():** Advances the token pointer and returns the consumed token.
- **Token &expect(TokenType t, const std::string &msg):** Ensures the next token matches the expected type, throwing an error if it doesn't.
- **bool check(TokenType t) const:** Checks if the next token matches the specified type.
- **bool match(TokenType t):** Consumes the next token if it matches the specified type.
- **bool atEnd() const:** Determines if the end of the token stream has been reached.
- **void skipNewlines():** Skips over any newline characters in the token stream.

- **ASTNodePtr parseStatement():** Parses a single statement from the token stream.
- **ASTNodePtr parseBlock():** Parses a block of statements enclosed in curly braces.
- **ASTNodePtr parseBodyOrStatement():** Parses either a block or a single statement, allowing for brace-optional syntax.
- **ASTNodePtr parseVarDecl(bool isConst):** Parses variable declarations, optionally marking them as constant.
- **ASTNodePtr parseFunctionDecl():** Parses function declarations, including their bodies.
- **ASTNodePtr parseClassDecl():** Parses class declarations, defining their structure and members.
- **ASTNodePtr parseIfStmt(), parseWhileStmt(), parseForStmt():** Parses control flow statements like `if`, `while`, and `for`.
- **ASTNodePtr parseReturnStmt():** Parses return statements, extracting the expression to be returned.
- **ASTNodePtr parsePrintStmt(), parseInputStmt():** Parses print and input statements, respectively.
- **ASTNodePtr parseCoutStmt(), parseCinStmt():** Parses output and input redirection statements using `cout` and `cin`.
- **ASTNodePtr parseImportStmt(bool isFrom = false):** Parses import statements, optionally specifying a module name.
- **ASTNodePtr parseExprStmt():** Parses expression statements.
- **ASTNodePtr parseCTypeVarDecl(const std::string &typeHint):** Parses variable declarations with a CType hint, supporting both primitive and pointer types.
- **bool isCTypeKeyword(TokenType t) const:** Checks if a token represents a CType keyword.

- **ASTNodePtr parseExpr():** Parses expressions using the Pratt parsing algorithm.
- **ASTNodePtr parseAssignment(), parseOr(), parseAnd(), ...:** Implement specific parsing rules for different operators and constructs, following the Pratt algorithm's principles.

- **ASTNodePtr parseArrayLiteral(), parseDictLiteral():** Parses literal array and dictionary constructions.
- **ASTNodePtr parseLambda():** Parses lambda functions, capturing variables and returning expressions.
- **ASTNodePtr parseArrowFunction(std::vector<std::string> params, int ln):** Parses arrow functions, similar to lambdas but with more explicit parameter handling.
- **std::vector<ASTNodePtr> parseArgList():** Parses argument lists for functions and lambdas.
- **std::vector<std::string> parseParamList(std::vector<bool> *outIsRef = nullptr, std::vector<ASTNodePtr> *outDefaultArgs = nullptr, std::vector<std::string> *outParamTypes = nullptr):** Parses parameter lists, identifying references, default arguments, and parameter types.

## Tradeoffs and Limitations

- **Flexibility vs. Complexity:** While the Pratt parsing algorithm offers great flexibility in handling operator precedence, it adds complexity to the implementation. Balancing between these factors required careful consideration.
- **Memory Usage:** Using references for token manipulation helps reduce memory overhead, but it also requires careful handling to avoid unintended side effects.
