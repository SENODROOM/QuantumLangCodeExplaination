# QuantumLanguage Compiler - Parser.h

## Overview

The `include/Parser.h` header file plays a pivotal role in the QuantumLanguage compiler by defining the parser class. The parser's primary function is to convert the sequence of tokens generated by the lexer into an Abstract Syntax Tree (AST). This AST represents the syntactic structure of the source code and serves as the foundation for subsequent compilation stages such as semantic analysis and code generation.

## Role in Compiler Pipeline

The parser operates within the lexical analysis phase of the compiler pipeline, following the lexer which breaks down the source code into individual tokens. The parser then takes these tokens and constructs a hierarchical representation of the code through the AST. This AST is crucial because it allows for easier manipulation and understanding of the code structure during later phases of compilation.

## Key Design Decisions and Why

1. **Exception Handling**: The parser throws custom `ParseError` exceptions when syntax errors are detected. These exceptions carry information about the error location (`line`, `col`) along with the error message, making debugging easier.

2. **Pratt-Style Precedence**: The parser uses a Pratt-parser approach for expression parsing. This method allows for easy handling of operator precedence without the need for recursive descent or complex lookahead rules. It simplifies the implementation and improves performance.

3. **Flexibility in Statement Parsing**: The parser supports both block statements and single statements, allowing for flexibility in how code can be structured. This is particularly useful for language features like brace-optional blocks.

4. **Support for Advanced Features**: The parser includes support for advanced language features such as lambda functions, arrow functions, and type hints. This ensures that the compiler can handle modern and expressive programming constructs.

## Major Classes/Functions Overview

### Parser Class

- **Constructor**: `explicit Parser(std::vector<Token> tokens)`
  - Initializes the parser with a vector of tokens.

- **parse() Method**: `ASTNodePtr parse()`
  - Converts the sequence of tokens into an AST.

### Token Helpers

- **current()**: `Token &current()`
  - Returns the current token being processed.

- **peek(int offset)**: `Token &peek(int offset = 1)`
  - Returns the token at a specified offset ahead in the token stream.

- **consume()**: `Token &consume()`
  - Consumes the current token and returns it.

- **expect(TokenType t, const std::string &msg)**: `Token &expect(TokenType t, const std::string &msg)`
  - Ensures the next token matches the expected type, throwing a `ParseError` otherwise.

- **check(TokenType t)**: `bool check(TokenType t) const`
  - Checks if the next token matches the given type.

- **match(TokenType t)**: `bool match(TokenType t)`
  - Consumes the next token if it matches the given type.

- **atEnd()**: `bool atEnd() const`
  - Determines if the end of the token stream has been reached.

- **skipNewlines()**: `void skipNewlines()`
  - Skips over any newline characters in the token stream.

### Parsing Methods

- **parseStatement()**: `ASTNodePtr parseStatement()`
  - Parses a single statement from the token stream.

- **parseBlock()**: `ASTNodePtr parseBlock()`
  - Parses a block of statements enclosed in curly braces.

- **parseBodyOrStatement()**: `ASTNodePtr parseBodyOrStatement()`
  - Parses either a block or a single statement, depending on the context.

- **parseVarDecl(bool isConst)**: `ASTNodePtr parseVarDecl(bool isConst)`
  - Parses a variable declaration, optionally marking it as constant.

- **parseFunctionDecl()**: `ASTNodePtr parseFunctionDecl()`
  - Parses a function declaration, including its parameters and body.

- **parseClassDecl()**: `ASTNodePtr parseClassDecl()`
  - Parses a class declaration, including its members and methods.

- **parseIfStmt()**, **parseWhileStmt()**, **parseForStmt()**: 
  - Implement parsing for conditional (`if`), loop (`while`, `for`), and control flow statements respectively.

- **parseReturnStmt()**: `ASTNodePtr parseReturnStmt()`
  - Parses a return statement, extracting the returned value if present.

- **parsePrintStmt()** and **parseInputStmt()**: 
  - Handle printing and input operations, respectively.

- **parseCoutStmt()** and **parseCinStmt()**: 
  - Manage output redirection using `cout` and input redirection using `cin`.

- **parseImportStmt(bool isFrom)**: `ASTNodePtr parseImportStmt(bool isFrom)`
  - Processes import statements, supporting both standard imports and those from specific modules.

- **parseExprStmt()**: `ASTNodePtr parseExprStmt()`
  - Parses expressions that are used as standalone statements.

- **parseCTypeVarDecl(const std::string &typeHint)**: `ASTNodePtr parseCTypeVarDecl(const std::string &typeHint)`
  - Handles variable declarations with explicit type hints, such as `int x = ...` or `int* p = ...`.

### Expression Parsing

- **parseExpr()**: `ASTNodePtr parseExpr()`
  - Parses a full expression using the Pratt-parser approach.

- **parseAssignment()**: `ASTNodePtr parseAssignment()`
  - Parses assignment expressions.

- **parseOr()**, **parseAnd()**: 
  - Implement logical OR and AND operators.

- **parseBitwise()**: 
  - Parses bitwise operators.

- **parseEquality()**: 
  - Handles equality and inequality comparisons.

- **parseComparison()**: 
  - Parses relational comparison operators.
