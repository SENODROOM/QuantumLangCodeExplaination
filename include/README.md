# QuantumLanguage Compiler - Compiler.h

## Overview

The `include/Compiler.h` header file is a fundamental part of the QuantumLanguage compiler, responsible for compiling abstract syntax trees (ASTs) into executable bytecode. This header defines the `Compiler` class, which encapsulates the logic for parsing and transforming source code into chunks of bytecode that can be executed by the virtual machine (VM).

## Role in Compiler Pipeline

The `Compiler` class plays a pivotal role in the compiler pipeline. It starts with parsing the source code into an AST using the parser module. Once the AST is constructed, the `compile` method of the `Compiler` class is invoked to traverse the AST and generate bytecode. The generated bytecode is then passed to the VM for execution.

### Key Design Decisions and Why

- **Use of Shared Pointers**: The `std::shared_ptr<Chunk>` is used to manage the lifecycle of bytecode chunks. This ensures that chunks are not prematurely deallocated while they are still being referenced by the VM.

- **Encapsulation of State**: The `CompilerState` structure encapsulates the current state of the compiler, including the chunk being compiled, local variables, upvalues, and scope depth. This allows the compiler to maintain context between different parts of the AST and handle nested scopes effectively.

- **Helper Functions**: Various helper functions like `emit`, `addConst`, and `addStr` simplify the process of emitting bytecode instructions and constants. These functions abstract away the details of bytecode manipulation, making the main compilation logic cleaner and more readable.

## Major Classes/Functions Overview

### Compiler Class

- **Constructor**: Initializes the compiler with an empty chunk and sets the initial scope depth to 0.
- **compile Method**: Compiles an entire program represented by an AST node and returns the top-level chunk containing the bytecode.

### CompilerState Structure

- **chunk**: A shared pointer to the current chunk of bytecode being compiled.
- **locals**: A vector of `Local` structures representing the local variables in the current scope.
- **upvalues**: A vector of `UpvalueDesc` structures representing upvalues, which are references to variables from outer scopes.
- **scopeDepth**: An integer indicating the current depth of the scope.
- **enclosing**: A pointer to the enclosing compiler state, allowing for nested scope handling.
- **isFunction**: A boolean flag indicating whether the current compilation is within a function.

### Helper Functions

- **emit**: Emits a bytecode instruction into the current chunk. If an operand is provided, it also emits the operand along with the instruction.
- **addConst**: Adds a constant value to the current chunk and returns its index.
- **addStr**: Adds a string literal to the current chunk and returns its index.
- **emitJump**: Emits a jump instruction into the current chunk and returns its index. The actual jump target is patched later.
- **patchJump**: Patches a previously emitted jump instruction with the correct offset based on the current position in the chunk.
- **beginScope** and **endScope**: Manage the beginning and end of new scopes, respectively. They adjust the scope depth and handle local variable declarations.
- **resolveLocal** and **resolveUpvalue**: Resolve the indices of local and upvalue variables based on their names and the current scope.
- **addUpvalue**: Adds a new upvalue to the current chunk and returns its index.
- **declareLocal**: Declares a new local variable in the current scope.

### Tradeoffs

- **Memory Management**: Using `std::shared_ptr` for managing bytecode chunks adds overhead but simplifies memory management by automatically deallocating chunks when they are no longer needed.

- **Scope Handling**: Encapsulating scope information within the `CompilerState` structure makes the compiler easier to understand and maintain, but it also increases the complexity of the codebase.

- **Performance vs. Readability**: The use of helper functions improves readability and reduces the complexity of the main compilation logic, but it might introduce some performance overhead due to additional function calls.

Overall, the `Compiler.h` file provides a robust framework for compiling QuantumLanguage programs into bytecode, leveraging encapsulation, shared pointers, and various helper functions to manage the compilation process efficiently.
