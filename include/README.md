# QuantumLanguage Compiler - Parser.h

## Overview

The `include/Parser.h` header file is an essential part of the QuantumLanguage compiler, focusing on the parsing phase. This file defines the `Parser` class which takes a vector of `Token`s generated by the lexer and constructs an Abstract Syntax Tree (`AST`) representing the syntactic structure of the source code. The `Parser` class handles the conversion of token streams into a structured format that can be easily interpreted and executed by the compiler.

This file plays a critical role in the compiler pipeline by translating the textual representation of the program into a hierarchical tree-like structure that represents the syntax of the language. This AST is then used by other phases of the compiler, such as semantic analysis, optimization, and code generation, to transform the program into machine-executable instructions.

## Key Design Decisions

### Error Handling

**Design Decision:** The parser uses exceptions to handle errors, specifically throwing a `ParseError` when a syntax error is encountered.

**Why:** Exceptions provide a robust mechanism for propagating errors up the call stack without requiring manual error checking at every level. This makes the code cleaner and more maintainable, allowing developers to focus on handling specific cases rather than checking error conditions repeatedly.

### Pratt Parsing Algorithm

**Design Decision:** The parser implements the Pratt parsing algorithm for expression parsing, which allows for flexible operator precedence rules and supports left recursion.

**Why:** Pratt parsing is particularly well-suited for languages with complex grammars and operators that have varying precedences. By using this algorithm, the parser can correctly interpret nested expressions and resolve operator priorities without the need for additional parsing passes or complex lookahead mechanisms.

## Class and Function Documentation

### ParseError

**Purpose:** A custom exception class derived from `std::runtime_error`, designed to store line and column information along with the error message.

**Behaviour:** When a syntax error occurs, a `ParseError` is thrown with details about the error location and message.

### Parser

**Purpose:** The main class responsible for parsing the input tokens into an AST.

**Constructor:** 
- `explicit Parser(std::vector<Token> tokens)`: Initializes the parser with a vector of tokens.

**Member Functions:**

#### parse()

**Purpose:** Parses the entire input stream and returns the resulting AST.

**Behaviour:** This function iterates through all tokens, constructing the AST based on the grammar rules defined in the parser.

#### current(), peek(), consume(), expect(), check(), match(), atEnd(), skipNewlines()

**Purpose:** Helper functions to manage token consumption and lookahead during parsing.

**Behaviour:** These functions facilitate the traversal of the token stream, enabling the parser to look ahead at upcoming tokens, consume tokens as they are processed, and handle different token types appropriately.

#### parseStatement(), parseBlock(), parseBodyOrStatement()

**Purpose:** Parses individual statements and blocks of statements.

**Behaviour:** These functions recursively construct the AST by breaking down the input into smaller components, handling control structures like if, while, and for loops, as well as variable declarations and function calls.

#### parseVarDecl(bool isConst)

**Purpose:** Parses variable declarations, including constant declarations.

**Behaviour:** This function creates an AST node representing a variable declaration, taking into account whether the variable is declared as constant.

#### parseFunctionDecl(), parseClassDecl()

**Purpose:** Parses function and class declarations.

**Behaviour:** These functions generate AST nodes for function and class definitions, including their parameters, return type, and body.

#### parseIfStmt(), parseWhileStmt(), parseForStmt(), parseReturnStmt(), parsePrintStmt(), parseInputStmt(), parseCoutStmt(), parseCinStmt(), parseImportStmt(bool isFrom = false), parseExprStmt()

**Purpose:** Parses specific control flow and statement types.

**Behaviour:** Each of these functions handles a particular type of statement, such as conditional execution (`if`), looping (`while`, `for`), returning from functions, printing, inputting values, and outputting formatted text.

#### parseCTypeVarDecl(const std::string &typeHint)

**Purpose:** Parses variable declarations with CType hints.

**Behaviour:** This function generates an AST node for a variable declaration, incorporating any CType hints provided in the source code.

#### isCTypeKeyword(TokenType t) const

**Purpose:** Checks if a given token type corresponds to a CType keyword.

**Behaviour:** This utility function determines whether a token type is one of the keywords associated with CType declarations.

#### parseExpr(), parseAssignment(), parseOr(), parseAnd(), parseBitwise(), parseEquality(), parseComparison(), parseShift(), parseAddSub(), parseMulDiv(), parsePower(), parseUnary(), parsePostfix(), parsePrimary()

**Purpose:** Implement Pratt parsing for various expression types.

**Behaviour:** These functions recursively build the AST for different types of expressions, respecting operator precedence and associativity rules.

#### parseArrayLiteral(), parseDictLiteral(), parseLambda(), parseArrowFunction(std::vector<std::string> params, int ln), parseArgList(), parseParamList(std::vector<bool> *outIsRef = nullptr, std::vector<ASTNodePtr> *outDefaultArgs = nullptr, std::vector<std::string> *outParamTypes = nullptr)

**Purpose:** Handle literal and function-specific constructs.

**Behaviour:** These functions create AST nodes for array literals, dictionary literals, lambda functions, arrow functions, argument lists, and parameter lists, respectively.
