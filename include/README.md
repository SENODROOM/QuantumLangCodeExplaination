# QuantumLanguage Compiler - Disassembler.h

## Overview

The `include/Disassembler.h` header file is an essential part of the QuantumLanguage compiler, focusing on the disassembly functionality. It defines functions to convert bytecode instructions into human-readable format, aiding in debugging and understanding the compiled code. This file plays a pivotal role in the compiler pipeline by providing insights into the internal workings of the generated bytecode.

## Key Design Decisions and Why

### Single Instruction Disassembly

The primary function, `disassembleInstruction`, is designed to pretty-print a single instruction from a bytecode chunk. Its simplicity ensures that each instruction can be analyzed individually, which is crucial for debugging purposes. The function always consumes one byte, reflecting the fixed-size nature of instructions in QuantumLanguage.

### Entire Chunk Disassembly

The secondary function, `disassembleChunk`, dumps the entire bytecode chunk. This comprehensive view helps in understanding the flow and structure of the program at a glance. By breaking down the chunk into individual instructions, it facilitates easier analysis and troubleshooting.

## Major Classes/Functions Overview

### disassembleInstruction

- **Purpose**: Converts a single bytecode instruction into a readable string representation.
- **Parameters**:
  - `const Chunk &chunk`: The bytecode chunk containing the instruction.
  - `size_t idx`: The index of the instruction within the chunk.
  - `std::ostream &out`: The output stream where the disassembled instruction will be written.
- **Return Type**: `size_t` indicating the number of bytes consumed by the instruction (always 1 in this context).

### disassembleChunk

- **Purpose**: Dumps the entire bytecode chunk, converting all instructions into a readable format.
- **Parameters**:
  - `const Chunk &chunk`: The bytecode chunk to be dumped.
  - `std::ostream &out`: The output stream where the disassembled chunk will be written.
- **Return Type**: `void`.

## Tradeoffs

### Simplicity vs. Flexibility

The decision to have `disassembleInstruction` consume exactly one byte simplifies its implementation and usage but limits its flexibility when dealing with variable-length instructions or complex formats. For QuantumLanguage's fixed-size instructions, this approach is straightforward and efficient.

### Readability vs. Performance

While the disassembler functions aim for readability, they could potentially impact performance during large-scale compilation or debugging sessions. However, given the relatively small size of bytecode chunks, these tradeoffs are minimal and do not significantly affect overall performance.

In conclusion, the `Disassembler.h` file is a vital tool for debugging and understanding the QuantumLanguage compiler's output. By providing clear and concise representations of bytecode instructions, it enhances the development process and aids in identifying issues more effectively.
