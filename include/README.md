# QuantumLanguage Compiler - Disassembler.h

## Overview

The `include/Disassembler.h` header file is an essential part of the QuantumLanguage compiler, responsible for disassembling bytecode into human-readable instructions. This process aids in debugging, understanding, and verifying the correctness of the compiled code at the assembly level.

## Role in Compiler Pipeline

The Disassembler plays a critical role in the QuantumLanguage compiler's pipeline by converting the intermediate representation (IR) generated by the compiler into a format that can be easily analyzed and understood. It operates after the compilation phase but before any execution occurs, allowing developers to inspect and validate the IR before it is executed on the target quantum hardware or simulator.

### Key Design Decisions and Why

1. **Single Instruction Pretty-Printing**: The function `disassembleInstruction` is designed to handle the pretty-printing of individual instructions. Given that each instruction in QuantumLanguage is represented by a single byte (`Opcode`), this function simplifies the output process, making it straightforward to display each opcode with its corresponding mnemonic.

2. **Entire Chunk Dumping**: The `disassembleChunk` function extends the functionality of `disassembleInstruction` by dumping the entire bytecode chunk. This comprehensive view helps in identifying patterns, errors, and optimizations within the compiled code.

3. **Stream-Based Output**: Both functions utilize `std::ostream` for outputting the disassembled instructions. This approach provides flexibility in directing the output to various destinations such as console, files, or other streams, enhancing usability and integration with different tools and environments.

## Major Classes/Functions Overview

1. **disassembleInstruction**:
   - **Parameters**:
     - `const Chunk &chunk`: A reference to the bytecode chunk containing the instructions.
     - `size_t idx`: The index of the current instruction within the chunk.
     - `std::ostream &out`: An output stream where the disassembled instruction will be written.
   - **Returns**: The number of bytes consumed by the instruction (always 1 for QuantumLanguage opcodes).
   - **Description**: This function takes a single instruction from the bytecode chunk and converts it into a readable format, writing it to the specified output stream.

2. **disassembleChunk**:
   - **Parameters**:
     - `const Chunk &chunk`: A reference to the bytecode chunk to be disassembled.
     - `std::ostream &out`: An output stream where the disassembled chunk will be written.
   - **Returns**: None.
   - **Description**: This function iterates through all instructions in the given bytecode chunk, calling `disassembleInstruction` for each one, and outputs the entire disassembled chunk to the specified stream.

## Tradeoffs

1. **Performance vs. Readability**: While disassembling improves readability and debugging capabilities, it also introduces performance overhead due to additional string manipulations and output operations. Balancing these factors ensures that the disassembler remains useful without significantly impacting the overall performance of the compiler.

2. **Flexibility vs. Simplicity**: Providing both single-instruction disassembly and chunk-level disassembly offers greater flexibility but increases complexity. Simplifying the interface might reduce flexibility but enhance ease of use for common tasks.

3. **Memory Usage**: Storing and processing large bytecode chunks requires careful memory management. Efficient algorithms and data structures help minimize memory usage while maintaining performance.

By addressing these tradeoffs, the Disassembler module contributes effectively to the QuantumLanguage compiler's development, offering valuable insights into the compiled code and facilitating robust debugging processes.
