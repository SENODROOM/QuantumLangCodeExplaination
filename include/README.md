# QuantumLanguage Compiler - Disassembler.h

## Overview

The `include/Disassembler.h` header file is a crucial component of the QuantumLanguage compiler, focusing on the disassembly functionality. This module is responsible for converting binary bytecode into human-readable assembly language, aiding in debugging and understanding the internal operations of the compiled programs.

## Role in Compiler Pipeline

In the QuantumLanguage compiler pipeline, the Disassembler plays a vital role during the debugging phase. It takes the bytecode generated by the compiler's other stages and converts it back into a readable format. This allows developers to inspect and understand the low-level instructions being executed by the virtual machine (VM).

## Key Design Decisions and Why

### Single Instruction Disassembly

The function `disassembleInstruction` is designed to handle the disassembly of individual instructions. Given the simplicity of the quantum language bytecode, each instruction occupies exactly one byte. This decision ensures that the disassembly process is straightforward and efficient, with minimal overhead.

### Entire Chunk Disassembly

The function `disassembleChunk` is used to disassemble an entire bytecode chunk. This function iterates through each instruction in the chunk, calling `disassembleInstruction` to convert them into their string representations. By providing both functions, the disassembler offers flexibility, allowing users to either inspect individual instructions or view the full disassembly of a chunk.

## Major Classes/Functions Overview

### `disassembleInstruction`

- **Purpose**: Converts a single bytecode instruction into its string representation.
- **Parameters**:
  - `const Chunk &chunk`: A reference to the bytecode chunk containing the instruction.
  - `size_t idx`: The index of the instruction within the chunk.
  - `std::ostream &out`: An output stream where the disassembled instruction will be written.
- **Return Value**: The number of bytes consumed by the instruction, which is always 1 for this compiler.

### `disassembleChunk`

- **Purpose**: Dumps the entire bytecode chunk as a disassembled assembly listing.
- **Parameters**:
  - `const Chunk &chunk`: A reference to the bytecode chunk to be disassembled.
  - `std::ostream &out`: An output stream where the disassembled chunk will be written.
- **Return Value**: None.

## Tradeoffs

### Simplicity vs. Flexibility

While the current design focuses on simplicity and efficiency, particularly with each instruction occupying only one byte, it may limit future scalability. For instance, if the bytecode were to expand to support more complex operations, additional logic would need to be added to handle variable-length instructions.

### Performance vs. Usability

The straightforward approach taken in `disassembleInstruction` prioritizes performance over usability. Since each instruction is handled individually, there is no need for complex parsing or formatting mechanisms. However, this might make the disassembly output less intuitive for users unfamiliar with the quantum language's opcode system.

By carefully considering these tradeoffs, the QuantumLanguage compiler aims to provide a balance between performance and usability, making it easier for developers to debug and understand their programs while maintaining optimal execution speed.
