# QuantumLanguage Compiler - Compiler.h

## Overview

The `include/Compiler.h` header file plays a pivotal role in the QuantumLanguage compiler by defining the core `Compiler` class. This class is essential for compiling source code into executable chunks, managing scopes, and handling various language constructs such as variables, functions, and classes. The VM described here ensures efficient and accurate execution of the program.

## Role in Compiler Pipeline

The `Compiler` class operates within the compiler pipeline, primarily during the compilation phase. It takes an abstract syntax tree (AST) generated by the parser and converts it into bytecode, which can be executed by the virtual machine. The class manages different stages of compilation, including:

- **Lexical Analysis**: Tokenizing the source code.
- **Parsing**: Constructing the AST from tokens.
- **Semantic Analysis**: Checking the AST for errors and verifying types.
- **Code Generation**: Translating the AST into bytecode.

## Key Design Decisions and Why

### Virtual Machine (VM)

The design includes a custom virtual machine (VM) that handles execution contexts, exception management, and runtime environments. This approach allows for fine-grained control over the execution process, enabling optimizations specific to quantum computing languages.

### Bytecode Representation

Bytecode is represented using a `Chunk` class, which stores the bytecode instructions along with their associated lines of source code. This representation facilitates debugging and error reporting by linking each instruction back to its original source location.

### Scope Management

Scopes are managed through nested `CompilerState` structures. Each `CompilerState` represents a lexical scope and contains information about local variables, upvalues, and the depth of the scope. This hierarchical structure helps in resolving variable names correctly across different scopes.

### Error Handling

Exception handling is integrated into the compiler pipeline, allowing for the generation of appropriate bytecode instructions when encountering errors. This ensures that the VM can handle runtime errors gracefully.

## Major Classes/Functions Overview

### Compiler Class

- **Constructor (`Compiler()`)**: Initializes the compiler with an empty chunk and sets the current compiler state.
- **compile (`std::shared_ptr<Chunk> compile(ASTNode &root)`)**: Compiles an entire program starting from the root AST node and returns the top-level chunk containing the bytecode.

### CompilerState Struct

- **chunk (`std::shared_ptr<Chunk>`)**: Holds the current chunk being compiled.
- **locals (`std::vector<Local>`)**: Stores information about local variables in the current scope.
- **upvalues (`std::vector<UpvalueDesc>`)**: Tracks upvalues used to capture variables from outer scopes.
- **scopeDepth (`int`)**: Indicates the depth of the current scope.
- **enclosing (`CompilerState *`)**: Points to the enclosing scope's state.
- **isFunction (`bool`)**: Flags whether the current scope is part of a function.

### Local Struct

- **name (`std::string`)**: Name of the local variable.
- **depth (`int`)**: Depth of the scope where the variable was declared.
- **isCaptured (`bool`)**: Indicates if the variable has been promoted to an upvalue.

### UpvalueDesc Struct

- **isLocal (`bool`)**: Specifies if the upvalue captures a local variable from the enclosing scope.
- **index (`int`)**: Slot index for the local or upvalue.

### Helper Functions

- **chunk (`Chunk &chunk()`)**: Returns a reference to the current chunk.
- **emit (`size_t emit(Op op, int32_t operand = 0, int line = 0)`)**: Emits a bytecode instruction into the current chunk.
- **addConst (`int32_t addConst(QuantumValue v)`)**: Adds a constant value to the chunk and returns its index.
- **addStr (`int32_t addStr(const std::string &s)`)**: Adds a string constant to the chunk and returns its index.
- **emitJump (`size_t emitJump(Op op, int line = 0)`)**: Emits a jump instruction and returns its index for later patching.
- **patchJump (`void patchJump(size_t idx)`)**: Patches a jump instruction with the correct offset.
- **beginScope (`void beginScope()`)**: Begins a new scope.
- **endScope (`void endScope(int line = 0)`)**: Ends the current scope and patches any unresolved jumps.
- **resolveLocal (`int resolveLocal(CompilerState *state, const std::string &name)`)**: Resolves a local variable within the given scope.
- **resolveUpvalue (`int resolveUpvalue(CompilerState *state, const std::string &name)`)**: Resolves an upvalue used to capture a variable from an outer scope.
- **addUpvalue (`int addUpvalue(CompilerState *state, int index, bool isLocal)`)**: Adds an upvalue to the current scope.
- **declareLocal (`void declareLocal(const std::string &name, int line = 0)`)**: Declares a local variable in the current scope.

### Statement Compilation Functions

- **compileNode (`void compileNode(ASTNode &node)`)**: Compiles a generic AST node.
- **compileBlock (`void compileBlock(BlockStmt &b)`)**: Compiles a block statement.
- **compileExpr (`void compileExpr(ASTNode &node)`)**: Compiles an expression.
- **compileVarDecl (`void compileVarDecl(VarDecl &s, int line)`)**: Compiles a variable declaration.
- **compileFunctionDecl (`void compileFunctionDecl(FunctionDecl &s, int line)`)**: Compiles a function declaration.
